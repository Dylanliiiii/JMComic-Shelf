# JMComic Shelf 桌面应用设计

## 目标

把旧的脚本式工作流整理为一个 Windows 桌面应用。第一版不重写上游下载能力，而是把现有下载、查看详情、PDF 转换、Markdown 总目录和本地书库浏览整合到一个可持续扩展的 UI 中。

桌面应用只管理当前设置里的一个下载目录。下载生成的 PDF、根目录 `Cover/` 封面缓存和 `catalog.md` 保存在用户选择的下载目录；漫画图片目录只作为下载与转 PDF 的中间产物，整理完成后不再保留。桌面端自己的 `settings.json`、`shelf.db` 和封面缩略图缓存在应用数据目录。

## 技术方向

- GUI：PySide6 + QFluentWidgets。
- 风格：深色 Fluent / Windows 11 Settings 风格。
- 强调色：青色，当前建议 `#00c8d7`。
- 下载能力：继续复用当前 `jmcomic` Python API、CLI、下载器和插件体系。
- 参考项目：`https://github.com/Dylanliiiii/LaunchDock`，只参考其 QFluentWidgets 桌面应用组织方式、深色 Fluent 风格、README 顶部呈现和项目协作说明。
- 打包与自动更新方向：`https://github.com/ok-oldking/pyappify`；GitHub Actions 打包参考 `https://github.com/ok-oldking/pyappify-action`。

## UI 原则

- 使用 `FluentWindow` 左侧导航，导航项包含图标和文字。
- 页面主区域背景保持透明或跟随 QFluentWidgets 主题。
- 不给全局 `QWidget` 强行设置白底或黑底。
- 不出现突兀的大块黑色矩形、白色画布、黑底白字说明条或网页后台式布局。
- 优先使用 QFluentWidgets 控件：`TitleLabel`、`SubtitleLabel`、`CaptionLabel`、`CardWidget`、`SmoothScrollArea`、`PrimaryPushButton`、`PushButton`。
- 设置页和列表页采用卡片式分组，整体紧凑、稳定、适合高频使用。
- 封面必须完整等比例显示，不裁剪、不截断。

## 页面结构

### 本地书库

本地书库页用于浏览当前下载目录中的本地漫画。

核心能力：

- 顶部搜索框支持输入 JM 号、作者名或标签，并通过短暂防抖避免输入法组合文字时频繁刷新列表。
- 默认筛选器为“全部”。
- “全部”左对齐显示，并带当前结果总数，外层使用轻量背景框突出当前筛选，同时保留青色下划线。
- “分类”按钮展开当前 SQLite 索引中出现过的全部标签；标签面板最多展示五行并支持滚动；可同时选择多个标签，满足任一选中标签的作品都会显示，点击“全部”回到完整列表。
- 从 SQLite 读取索引前，先递归扫描当前设置的下载目录，把现有 PDF、根目录 `Cover/` 封面缓存和旧版漫画图片目录同步到 SQLite。
- 按第一作者分组展示作品。
- 作品以封面卡片展示，封面在上，底部显示 `JM号 + 标题` 的省略文本。
- 封面卡片网格根据窗口宽度自动调整列数，不固定每行 5 本。
- 左侧导航展开或收起导致内容宽度变化时，应延迟重排封面网格，避免在本地书库页反复切换侧边栏时出现明显卡顿。
- 从其他页面切回本地书库时，应先显示现有 SQLite 索引内容，再在后台扫描下载目录；目录扫描完成后再刷新列表，避免页面切换被同步扫描阻塞。
- 点击封面打开 PDF。
- 右键封面提供“打开 PDF”和“在文件资源管理器中显示位置”。
- 批量管理模式下，封面卡片左上角显示选择框；点击选择框或整张卡片可切换选中状态。
- 批量管理提供全选、反选、取消全选和删除选中；删除前必须弹出确认提醒，说明会删除所选漫画的本地文件和桌面端索引。

“全部”是桌面端内置筛选器，不写入真实标签。
标签统一保存为中文简体；标题、作者和章节标题保持来源文本。

### 禁漫下载

禁漫下载页作为桌面端下载入口，旧的下载双击脚本不再保留。

核心能力：

- 输入一个或多个 JM 号，支持空格、换行、英文逗号和中文逗号分隔。
- 开始下载后，所有 JM 号先进入任务列表，输入框随即清空，状态从“等待中”更新为“下载中”“已完成”或“失败”。
- 第一版提供任务级总进度和当前任务提示；上游下载器未提供图片级回调时，不伪造图片级百分比。
- 所有任务结束后，进度条应归零，并显示“全部已完成”或失败数量；任务表格继续保留成功和失败记录。
- 使用当前设置中的 `jmcomic-option.yml` 和下载根目录。
- 下载完成后继续生成 PDF。
- PDF 生成后整理为 `下载根目录 / 第一作者 / JM号-作品名.pdf`，作者目录下不再保留每本漫画的图片文件夹。
- 下载根目录下的 `Cover/` 缓存每本漫画第一面图片，供书库封面读取。
- 下载完成后继续更新下载根目录下的 `catalog.md`，多作者作品只登记到第一作者下。
- 下载完成后同步更新 SQLite 书库索引。
- 失败任务保留错误摘要，并提供重试入口。

### 禁漫预览

禁漫预览页作为桌面端查看详情入口，旧的预览双击脚本不再保留。

核心能力：

- 页面标题单独成行，JM 号输入框和查看按钮位于标题下方。
- 输入单个 JM 号。
- 点击“查看详情”或在输入框按 Enter 都会查询。
- 查询期间显示搜索中状态，并暂时禁用输入框和按钮，避免误以为卡住或重复触发。
- 复用当前 `jmcomic` 查看详情能力。
- 在文字详情上方显示完整等比例封面图；本地索引已有封面时优先显示本地封面，缺失时缓存线上封面。
- 展示标题、作者、标签、章节、链接等信息。
- 查询前同步扫描下载目录，以便识别本地 PDF。
- 如果作品已下载，提供打开 PDF 和定位文件入口。

### 设置

设置页管理桌面端运行所需的本地路径。

第一版设置：

- 当前下载目录。
- `jmcomic-option.yml` 路径。
- 应用数据目录，只读显示。
- 外观主题：跟随系统、浅色、深色。
- 清理封面缓存按钮。
- 重建索引按钮。

保存设置时，如果选择了配置文件和下载目录，应同步更新 `jmcomic-option.yml` 中的 `dir_rule.base_dir`。

## 数据存储

桌面端主数据源使用 SQLite，不使用 `catalog.md` 作为唯一结构化索引。
SQLite 中的标签与 `catalog.md` 一致，统一转换为中文简体后保存。

默认位置：

```text
%APPDATA%/JMComic Shelf/
  settings.json
  shelf.db
  covers/
```

下载目录仍保存用户内容：

```text
下载目录/
  作者/
    JM号-作品名.pdf
  Cover/
    JM号-作品名.jpg
  catalog.md
```

## 索引重建

第一版必须支持从下载目录重建索引。

扫描规则：

- 识别 `作者 / JM{Aid}-{Atitle}.pdf`。
- 识别根目录 `Cover/JM{Aid}-{Atitle}.jpg` 作为封面来源。
- 识别旧版 `作者 / JM{Aid}-{Atitle} / 第{Pindex}章 / 图片`。
- 识别下载根目录或子目录中的 `JM{Aid}-{Atitle}.pdf`。
- 合并相同 JM ID 的目录信息和 PDF 路径。
- 生成桌面端封面缩略图，缩略图完整等比例，不裁剪。
- 重建完成后，SQLite 书目集合和 `catalog.md` 都应按本次实际扫描到的本地漫画同步裁剪，清除已经没有 PDF 或图片目录的旧条目。

## 错误处理

- 下载目录不存在：设置页提示重新选择。
- `jmcomic-option.yml` 不存在：下载页禁用或显示中文错误。
- PDF 文件不存在：封面卡片仍可显示，但打开 PDF 动作禁用或提示缺失。
- 数据库不可读：书库页显示错误卡片，不让应用启动失败。
- 封面缓存缺失：使用占位封面，并可重建缓存。

## 技术栈与限制

- 当前桌面端是 PySide6 + QFluentWidgets，不是原生 WinUI 3；后续 UI 修改必须遵守 QFluentWidgets 的主题和控件体系。
- 仓库内部分历史中文文档和测试文本曾出现 mojibake 乱码；修改到的文件必须恢复正常 UTF-8 中文。
- 当前目录扫描是同步执行；下载目录变大后应改为后台线程或增量索引。
- 书库页不能再只依赖下载完成后的 SQLite 写入；必须保留从现有下载目录重建的能力。

## 非目标

第一版不做：

- 多下载目录聚合。
- 在线账号管理复杂向导。
- 完整账号向导和复杂更新设置页。
- 重写上游下载器。
- 删除 `catalog.md`。

## Release 与打包

- 使用 `pyappify.yml` 描述 PyAppify 打包入口。
- 使用 `icons/icon.png` 和 `icons/icon.ico` 作为 PyAppify 打包图标，来源为 `assets/icon.png`。
- 推送 `v*` tag 时，由 `.github/workflows/release.yml` 调用 `ok-oldking/pyappify-action` 构建 Windows 包并上传到 GitHub Release。
- 版本号使用 `vMAJOR.MINOR.PATCH`，例如 `v0.1.0`。
- 暂不配置 CNB 镜像或同步 Action。
