# JMComic Shelf 桌面应用设计

## 目标

把当前依赖 `download-jmcomic.bat` 和 `view-jmcomic.bat` 的脚本式工作流，整理为一个 Windows 桌面应用。第一版重点不是重写上游下载能力，而是把现有下载、查看详情、PDF 转换、Markdown 总目录和本地书库浏览整合到一个可持续扩展的 UI 中。

桌面应用面向当前个人本地书库场景：只管理当前设置里的一个下载目录，并只更新这个下载目录。

## 技术方向

- GUI：PySide6 + QFluentWidgets。
- 风格：Windows 原生 Fluent 风格，参考 `ok-oldking/ok-script` 的工具型应用体验。
- 下载能力：继续复用当前 `jmcomic` Python API、CLI、下载器和插件体系。
- 打包方向：后续研究 `ok-oldking/pyappify`，不默认使用 PyInstaller；第一版 GUI 不把自动更新作为核心阻塞项。

## 应用结构

采用方案 A：左侧导航工作台。

左侧导航使用“图标 + 文字”的功能入口，不使用只有图标的窄栏。第一版入口：

- 书库
- 下载
- 查看详情
- 设置

默认进入“书库”页。

## 书库页

书库页用于浏览已经下载并写入索引的漫画。

核心能力：

- 顶部搜索框支持输入 JM 号、作者名或标签。
- 默认筛选器为“全部”，展示当前下载目录下所有已下载作品。
- 输入 JM 号时，只展示对应作品。
- 输入作者名时，展示匹配作者，并在作者分组下列出该作者作品。
- 输入标签时，展示满足标签的作品，并按作者分组；每个作者标题显示作者名和命中作品数量。
- 作品以封面卡片展示：封面图在上，底部显示 JM 号和标题。
- JM 号和标题不完整铺开，使用类似文件夹中文件名的省略显示，适配封面宽度。
- 单击或双击封面时提供轻量反馈动画，然后打开对应 PDF。
- 右键封面打开上下文菜单，提供“打开 PDF”和“在文件资源管理器中显示位置”。

“全部”是桌面端内置筛选器，不写入真实标签。

## 下载页

下载页替代 `download-jmcomic.bat`。

核心能力：

- 输入一个或多个 JM 号，支持空格、换行或逗号分隔。
- 使用当前设置中的 `jmcomic-option.yml` 和下载根目录。
- 下载完成后沿用当前目录结构，接近 `作者 / JM号-作品名 / 第N章`。
- 下载完成后继续生成 PDF。
- 下载完成后继续更新下载根目录下的 `catalog.md`。
- 下载完成后同步更新 SQLite 书库索引。
- 下载任务应显示基本状态：等待、下载中、成功、失败。
- 失败任务应保留错误摘要，便于后续重试或排查。

第一版可以串行或使用现有配置中的并发能力，不单独设计复杂任务队列。

## 查看详情页

查看详情页替代 `view-jmcomic.bat`。

核心能力：

- 输入单个 JM 号。
- 复用当前 `jmv` / `jmcomic` 查看详情能力。
- 在桌面端展示标题、作者、标签、章节、链接等信息。
- 如果该作品已下载，展示本地 PDF 状态，并提供打开 PDF 和显示位置入口。
- 如果未下载，提供跳转到下载页并带入 JM 号的入口。

## 设置页

设置页管理桌面端运行所需的本地路径和用户偏好。

第一版设置：

- 当前下载目录。
- `jmcomic-option.yml` 路径。
- 应用数据目录，默认 `%APPDATA%/JMComic Shelf/`。
- 封面缓存清理按钮。
- 重新扫描/重建索引按钮。

下载目录由用户自定义。应用数据目录默认使用 Windows 用户数据目录，程序安装目录只放应用代码，避免自动更新覆盖用户数据。

后续可以提供高级选项，把应用数据目录迁移到下载盘；第一版只需要保留设计余地。

## 数据存储

桌面端主数据源使用 SQLite，不使用 `catalog.md` 作为结构化索引。

默认位置：

```text
%APPDATA%/JMComic Shelf/
  shelf.db
  settings.json
  cache/covers/
```

下载目录仍保存用户内容：

```text
下载目录/
  作者/
    JM号-作品名/
      第N章/
  JM号-作品名.pdf
  catalog.md
```

SQLite 建议表：

- `settings`：轻量键值设置，或由 `settings.json` 承担。
- `albums`：JM ID、标题、链接、PDF 路径、封面缓存路径、下载时间、更新时间。
- `authors`：作者名。
- `tags`：标签名。
- `album_authors`：作品与作者的多对多关系。
- `album_tags`：作品与标签的多对多关系。
- `chapters`：章节 ID、序号、标题、所属作品。

第一版只管理一个下载目录，因此不需要多 `library_roots` 表。后续如果要支持多个书库，再引入 `library_roots`。

## Markdown 总目录

`catalog.md` 继续保留，并继续放在用户下载根目录下。

它的定位是人能直接打开查看的总目录，不是桌面端主索引。下载完成后应同时更新：

- SQLite：给桌面端搜索、筛选、打开 PDF 和定位文件使用。
- `catalog.md`：给用户作为独立文件查看下载历史。

## 封面缓存

封面缓存只保存桌面端需要的缩略图，不保存原图。

缩略图规则：

- 必须完整保留封面画面。
- 只能等比例缩小和适度压缩画质。
- 不允许裁剪、截断或取局部。
- 卡片显示时如比例不一致，使用完整图居中显示，空白区域由 UI 背景承担。
- 缓存可清理、可重建。

## 数据流

### 下载完成

1. 用户在下载页输入 JM 号。
2. 桌面端调用现有下载能力。
3. 下载器按当前配置保存图片。
4. 插件生成 PDF。
5. 插件或桌面端索引服务更新 SQLite。
6. `CatalogPlugin` 更新 `catalog.md`。
7. 书库页刷新索引结果。

### 启动应用

1. 读取 `settings.json`。
2. 打开或创建 `shelf.db`。
3. 检查当前下载目录是否存在。
4. 加载“全部”作品列表。
5. 如果数据库为空，提示用户下载作品或从现有 `catalog.md` / 下载目录重建索引。

### 重建索引

第一版应提供重建入口。重建策略优先从 SQLite 已知路径和下载目录扫描 PDF 文件；如需要补充作者、标签等元数据，可解析 `catalog.md` 作为辅助来源。

## 错误处理

- 下载目录不存在：设置页提示重新选择。
- `jmcomic-option.yml` 不存在：下载页禁用下载按钮，并提示到设置页配置。
- PDF 文件不存在：封面卡片仍可显示，但打开 PDF 按钮禁用或提示文件缺失。
- 数据库损坏：提示备份损坏文件并重建索引。
- 封面缓存缺失：使用占位封面，并后台重建。
- 下载失败：保留失败 JM 号、错误摘要和重试入口。

## 测试策略

设计阶段后进入实现时，优先覆盖：

- SQLite 索引写入和去重。
- 作者、标签、JM 号筛选。
- “全部”筛选不污染真实标签。
- 多作者作品在各作者分组下展示。
- PDF 路径缺失时的错误状态。
- 封面缩略图完整等比例生成，不裁剪。
- 下载完成后同时更新 SQLite 和 `catalog.md`。

## 非目标

第一版不做：

- 多下载目录聚合。
- 在线账号管理复杂向导。
- 完整自动更新系统。
- 正式 release/tag。
- 重写上游下载器。
- 删除 `catalog.md`。

## 待用户确认

- 查看详情页是否需要保存查询历史。
- 设置页是否第一版就暴露“应用数据目录迁移”，还是先只显示当前路径和清理缓存。
